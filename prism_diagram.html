PRISM Pipeline

End-to-end research intelligence: ingest ArXiv papers, hybrid retrieval, structured briefs, and cross-document conflict detection.

ArXiv API pdfplumber BM25 + Dense Qdrant Groq LLM Phase 3: LangGraph Python
1
ArXiv PDF Fetch
Queries ArXiv API via feedparser for papers matching a topic. Downloads PDFs with SHA-256 dedup — already-ingested papers are skipped. Extracts title, authors, abstract, and published date as metadata.
feedparser ArXiv API SHA-256 dedup src/ingestion/fetch.py
2
PDF Parse & Normalize
Extracts raw text from PDFs using pdfplumber. Groups text blocks by vertical page position to preserve reading order. Strips headers, footers, and reference noise. Outputs clean normalized text per paper.
pdfplumber vertical grouping src/ingestion/parse.py
3
Overlapping Window Chunker
Splits normalized text into overlapping windows. Each chunk is 1000 characters with a 200-character overlap and an 800-character step. Overlap preserves cross-boundary context so retrieval doesn't miss sentences that straddle chunk edges.
chunk_size=1000 overlap=200 step=800 src/ingestion/chunk.py
4
Dense Embedding
Each chunk is encoded with a sentence-transformers model into a dense vector. The model is lazy-loaded and cached in _MODEL_CACHE — subsequent papers skip re-initialization. Vectors are stored in Qdrant with chunk text and paper metadata as payload.
sentence-transformers model cache Qdrant upsert src/embeddings/embed.py
Hybrid Retrieval (Phase 2)
BM25 Sparse Search
Loads all chunks from Qdrant, tokenizes, builds a rank_bm25 index. Returns top-k scored chunks for exact keyword matches. Fast, no GPU needed.
rank_bm25 src/retrieval/bm25.py
Dense Vector Search
Encodes the query into a vector. Qdrant cosine similarity search via .query_points() returns top-k semantically similar chunks.
Qdrant cosine src/store/qdrant.py
RRF Fusion (Reciprocal Rank Fusion)
Combines BM25 and dense ranked lists using RRF scoring: score = Σ 1/(k + rank_i) where k=60. De-duplicates chunks and re-ranks by fused score. Best of both: exact + semantic.
RRF k=60 src/retrieval/hybrid.py
6
Groq LLM — Structured Intelligence Brief
Top fused chunks are passed to Groq with a JSON-mode prompt. Generates a structured brief: key findings, methodology, limitations, and implications. max_tokens=4096. Hallucination guard: empty or incoherent sections are silently skipped rather than fabricated.
Groq JSON mode max_tokens=4096 hallucination skip src/output/brief.py
7
Cross-Document Conflict Detection
Compares key findings across all ingested papers. Flags pairs of contradicting statements — e.g., one paper claims X improves performance while another finds no significant effect. Outputs a conflict report alongside the briefs.
cross-doc compare conflict flags src/output/conflicts.py
Phase 3 — Planned
LangGraph Multi-Agent Orchestration
Replaces the linear orchestrator with a LangGraph state machine. Specialized agents — Ingestion, Retrieval, Synthesis, Conflict, Citation — run as coordinated graph nodes. Enables conditional branching (re-retrieve if confidence is low), parallel ingestion of N papers, and a persistent agent memory layer.
LangGraph state machine multi-agent parallel ingestion agent memory
External Services & Dependencies
ArXiv API
Free academic paper search + PDF download. feedparser parses the Atom feed. No API key needed.
Qdrant (Local)
Local vector store. Cosine similarity search over chunk embeddings. No cloud dependency in Phase 1–2.
Groq LLM
LLM inference for structured briefs and conflict detection. JSON mode, fast inference, large context window.
sentence-transformers
Local embedding model. Cached after first load. No API calls required — fully offline-capable.
rank_bm25
BM25 sparse index built in-memory over all stored chunks. Rebuilt each session; stateless between runs.
Secrets (.env)
GROQ_API_KEY, GROQ_BASE_URL, embedding model name, Qdrant settings. Never committed — .gitignored.
